**命令行工具选择**：
在Windows上，可使用 **Git Bash** 以避免路径和环境变量问题。
注意在Git Bash中，**磁盘路径应写为 `/d/MyProject/` 形式，而非 `D:\MyProject\`。**

### 克隆远程仓库
1. 在电脑上选取（新建）一个用于存放该仓库内容的文件夹，打开
2. 在此处打开终端
3. 执行克隆命令： `git clone 仓库的网址`，这会在当前文件夹下创建一个与仓库同名的文件夹

### 提交并推送本地更改
1. 添加更改：`git add 文件名`  
   - 添加当前目录及所有子目录下的所有更改（新增、修改、删除）：`git add .`  
     > **说明**：`git add .` 会递归处理子文件夹，不是仅限当前文件夹一层。  
     > 如果只想添加当前文件夹下的文件（不包含子文件夹），需使用路径限制，如 `git add *.txt`。
2. 提交更改：`git commit -m "对本次更改的注明"`
3. 推送上线：`git push origin main`(`origin` 是远程仓库的默认别名，`main` 是分支名)

注：
**工作区与暂存区**：工作区即本地文件夹；`git add` 将更改存入暂存区；`git commit` 将暂存区内容提交到本地仓库。

### 查看状态与历史

1. `git status`：查看工作区和暂存区的文件状态（哪些修改了、哪些已暂存）。
2. `git log`：查看详细的提交历史，按时间倒序排列。
	- `git log --oneline`：查看简洁版的提交历史（仅显示提交哈希和说明）。
    - `git log --graph`：以图表形式查看分支合并历史。

### 同步远程更改
- `git pull origin main`：用于从远程仓库`origin`的`main`分支拉取最新更新并合并到本地当前分支。（如果遇到冲突，需要手动解决）**这是开始本地工作前的推荐习惯**

### 分支基础操作
1.  `git branch`：查看所有本地分支（当前分支前会有 `*` 号标记）
2.  `git checkout -b 新分支名`：创建并切换到一个新分支。尝试重大改动时使用，不影响主分支。
3.  `git checkout 分支名`：切换到已存在的分支。
4.  `git branch -d 分支名`：删除已合并的分支。
